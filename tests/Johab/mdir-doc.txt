   Mdir III는
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

  Mdir III는 프로그램을 편하게 실행하는데 주목적이 있으며, 아울러 파일과 디렉
  토리, 압축파일을 쉽게 관리할 수 있도록 도와주는 프로그램입니다.
  제작자는 다음과 같은 생각으로 프로그램을 만들었습니다.

  o 컴퓨터를 전혀 모르는 사람도 쉽게 사용하게끔 기본적인 사용법이 간단합니다.
    영문메시지를 사용한다는 점만  제외하면  정말로 사용하기 쉬운 도스셸이라고
    감히 말씀드립니다. (한글도스를 사용할 경우 한글메세지가 표시됩니다.)
    아울러 컴퓨터를 어느 정도  다룰 줄 아는 분들에게는 작업 속도를 높여드릴것
    입니다. 기본적으로 화살표키, 엔터키, Alt키, Ctrl키, F10키만 알고 나머지는
    화면하단의 정보를 참고하면 됩니다.  Alt키나 Ctrl키를 살짝 눌러보십시오.
    극소수의 고급 사용자를 위한 기능은 아예 넣지를 않았습니다.  가능하면 쉽게
    사용할 수 있도록 꼭 필요한 기능들로만 모아두었습니다.

  o 자주 사용하는 프로그램을 펑션키에 등록하여 편하게 실행할 수 있습니다.  이
    것은 컴퓨터 사용에 있어서 최대한의 효율을 제공할 것입니다. 총 28개의 펑션
    키에 각기 다른 프로그램을 지정할 수 있으며, 하나의 펑션키에 여러개의 프로
    그램을 등록하는 것도 가능합니다. 자주 사용하는 프로그램은 원터치로~~

  o 압축파일을 쉽게 관리할 수 있습니다.  흔히 사용하는 대부분의 압축파일을 지
    원하는 것은 물론 압축파일속의 파일까지도 간단히 실행시키고 볼수 있습니다.
    압축파일을 만들고, 풀고, 지우는 기본적인 기능이지만 전문 압축파일관리기에
    못지 않게 편리하고 사용하기 쉽게 만들었습니다.  통신을 하거나 자료를 백업
    하는등 압축파일을 사용하는 빈도가 많아지므로  무엇보다 유용한 기능이 될것
    같습니다.  또한 "자동풀림압축파일"도 지원하며, 확장자를 잘못준 압축파일도
    별다른 메뉴를 호출할 필요없이 스스로 알아서 처리합니다. 귀찮은것도 쉽게~

  o 단순한 아름다움을 추구합니다. 화려하고 요란하게 구성할 수도 있지만 오히려
    눈에 잘 들어오지 않아서 장시간 사용시 효율이 떨어지므로 간단한 화면구성에
    서 최대한 깔끔하게 만들려고 노력했습니다. 이왕이면 깔금한 화면에서~~

  o 프로그램을 실행하는 방법이 다양하고 제공하는 사항도 많습니다.  도스셸이라
    면 무엇보다도 프로그램을 실행하는 기능이 강력해야 합니다. 그리고 프로그램
    실행시 점유메모리가 적어야 합니다.  "M3"는 메모리를 아예점유하지 않을수도
    있으며 스와핑을 하면 아주 적은 메모리만 사용하고 나머지는 실행되는 프로그
    램이 사용할 수 있도록 해줍니다. 도스셸의 제약을 넘어서~~

  o 사용자가 색깔을 지정할 수 있습니다.  아울러 파일의 확장자별로 색깔을 달리
    줄수도 있어서 실행파일등을 한눈에 파악할 수 있습니다. 색깔을 마음대로~~

   <주> (*)표시가 되어 있는 파일은 필요시 생성되었다 자동으로 지워지는 파일입
        니다.   위 파일이 지워지지 않고 남아있는 경우는 해당작업이 정상적으로
        종료되지 않아서 지우지 못한 상태이니  이상하게 디스크의 용량이 줄어드
        는 경우 확인해 보시고 지우세요.   단, XMS나 EMS메모리가 충분한 경우는
        디스크로 스와핑을 하지 않으므로 이런 현상은 없습니다.
        스와핑루틴은 랄프 브라운(Ralf brown)의 SPAWNO를 사용했습니다.

        _M_.BAT파일과 (M).BAT파일은 프로그램 실행중 지우지 마시기 바랍니다.
        지울경우 원하지 않은 결과를 초래할 수 있습니다.

   <참고> 압축파일을 처리하다 보면 이름이 '(M)'인 디렉토리가 만들어져 있을 수
          있습니다. 무시하고 계속 사용하면 디렉토리는 자동으로 지워지게 되어
          있습니다.

   M3의 실행
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

  M3를 실행하는 방법을 알아보자. 이해하기 힘든 분은 도스상에서 M만 입력하고
  엔터치면 실행이 된다는 점만 기억하고 다음 부분으로 넘어가시기를...

  M [/선택사항] [파일명] [디렉토리명] [/선택사항]

  []로 둘러싼 것은 생략이 가능하다는 의미이다.  따라서 전부 생략하더라도 실행
  은 된다. 단지, 필요에 의해서 여러가지 선택사항을 덧붙일수 있다는 얘기다.
  평소엔 M만 입력한후 엔터키를 누르면 된다.                      ( 예: m ← )

    1. /로 시작하는 것은 옵션으로 인식한다.                        ( 예: /? )
    2. '*','?',';'문자가 포함될 경우는 파일명으로 인식한다.     ( 예: *.DOC )
    3. 1도 아니고 2도 아닐경우는 디렉토리명으로 인식한다.         ( 예: dos )

  M3에서는 패스명의 일부분으로도 원하는 곳으로 이동할 수 있다.  만약 I로  시작
  하는 디렉토리가 하나 밖에 없다면 M I만으로도 이동할 수 있다.

  디렉토리를 찾을때는
    1. 해당 이름과 완전히 똑같을 경우
    2. 처음몇글자가 동일할 경우
    3. 현재 디렉토리

  이때 찾은 디렉토리가 여러개 일경우는 디렉토리 관리기 화면안으로 들어가서 빨
  리 찾기 기능이 작동되게 된다. 스페이스키를 누르면 다음 디렉토리를 보여준다.

  (예)  M i    -> i*.*   디렉토리로 이동한다.
        M dow  -> dow*.* 디렉토리로 이동한다.


☞ M3는 시작할 때 프로그램의 여러가지 선택사항을  지정해 두는 파일인 M.CFG 를
   읽어 들이게 된다.  만약 여러개의 환경설정 파일을 작성해 두고 사용할 경우엔
   다음과 같은 방법으로 이용할 수도 있다.

   SET M_CFG=환경설정 파일의 완전한 경로명  (예) SET M_CFG=c:\mdir\m2.cfg

   선택사항(Command line option)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

  옵션이나 사용환경등을 간단히 지정할 수 있다. 환경설정으로 환경을 모두 지정해
  두고 사용할 수 있으나  필요시 적절히 바꿜볼 수 있도록 아래와 같은 명령행옵션
  을 준비해 두었다. 이해하기 힘든 분은 이 부분을 건너 띄어 무방하다.

  ┌──────┬─────────────────────────────┐
  │?           │도움말.                                                   │
  │1,2,3,4,5,6 │한줄에 파일을 1,2,3,4,5,6개씩 출력.                       │
  │N,E,S,T     │파일명,확장자,크기,시간순으로 정렬.                       │
  │J,K,!       │조합형, 완성형 선문자, 한글바이오스자동감지.              │
  │D           │디렉토리와 실행파일만 표시함.                             │
  │L           │파일명을 소문자로.                                        │
  │A           │Hidden파일 포함+속성보기.                                 │
  │Z           │Hidden파일 포함.                                          │
  │V           │EGA/VGA의 28라인(대문자), 43/50라인(소문자) 사용함.       │
  │+           │키입력 속도를 빠르게.  (AT이상)                           │
  │C           │MCD부터 시작함. (대문자:그냥종료함, 소문자:작업계속)      │
  │R           │디렉토리구조를 새로 읽어서 저장한 후 시작함.              │
  │P           │DIR명령모드. (소문자:디렉토리 바꾸어줌)                   │
  │=           │메인화면의 파일리스트화면의 크기를 고정시킴.              │
  │X           │풀메모리 실행을 작동하지 않게 함. 스와핑으로 실행하게 됨  │
  └──────┴─────────────────────────────┘

  <주의> /p나 /P옵션이 사용되면 /V나 /v(화면모드바꾸기)가 작동하지 않는다.
         28라인이상을 사용하다 M을 종료하게 되면 화면을 깨끗이 지우게 된다.

         /p옵션은 디렉토리명을 주면 해당 디렉토리를 보여준후 디렉토리를 바꿔
         준다.   즉, 'm /p dos' 라고 치면 'dos'라는 디렉토리를 찾아 보여주고
         디렉토리를 이동한다.


  (사용예)
  M           <- 그냥 실행한다.
  M /?        <- 도움말을 본다.
  M /m        <- 색깔번호표를 본다.
  M *.exe     <- *.EXE 파일을 보여준다.
  M ;m.exe    <- M.EXE 파일만 보여준다.
  M .gif;.doc <- *.GIF와 *.DOC를 보여준다.
  M dow       <- dow란 디렉토리로 이동. 없으면 비슷한 이름의 디렉토리로 이동.
  M dow *.exe <- dow란 디렉토리로 이동하여 *.exe파일을 보여준다.
  M /4z c:\   <- c:\에 있는 모든속성의 파일을 4개씩 출력.
  M /V        <- 28라인모드로 전환함.    (대소문자 유의)
  M /v        <- 43/50라인모드로 전환함.
  M /c        <- 초기화면이 메인화면대신 MCD부터 먼저 나온다. (대소문자 유의)
  M /C        <- NCD대체 프로그램 사용할때.
  M /cp       <- NCD대체 + DIR대체 겸용
  M /CP       <- M /cp 혹은 M /Cp와 동일함
  M /p        <- DIR대체 프로그램으로 사용할때.

  o M /e/a/l 와 M/E/A/L 와 M /eal 는 같은의미로 해석된다.
  o M /e *.* /al 와 M /eal *.* 와 M *.* /eal 도 같은의미로 해석된다.
  o 옵션은 /로 시작하며, 공백없이 엮어서 사용할수 있다.
  o 동일한 항목이 여러번 나올 경우는 뒤쪽이 유효하다.

  <참고>
  o MDIR.BAT는 항상 /p옵션을 붙여서 실행하는  배치파일로써  디렉토리의목록만을
    보여주고 종료하므로, 도스의 DIR명령을 흉내낼수 있다.
  o MCD.BAT는 항상 /cp옵션을 붙여서 실행하는 배치파일로써,  NCD대체용으로 사용
    할 수 있다. /cp대신에 /C를 사용하면 종료시 DIR화면을 보여주지 않게 된다.


  (사용예)
  MDIR        <- 현재 디렉토리의 파일목록을 보여준다.
  MDIR dos    <- dos  디렉토리의 파일목록을 보여주고 디렉토리를 이동한다.
  MDIR /c     <- 디렉토리를 선택한 후 목록을 보여준다.
  MDIR dos /d <- dos디렉토리의 실행파일만 보여준다.

  MCD         <- MCD를 부른다.
  MCD dos     <- dos 디렉토리의 파일목록을 보여준다.
  MCD /r      <- 디렉토리구조를 디스크에서 다시 읽어온다.

   문장입력(String input)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

  파일명이나 명령등을 입력하는 곳에서의 편집글쇠 사용방법을 알아보자.

  초기치에 수정할 경우 : Home키 또는 좌/우 화살표키나 스페이스키를 누른다.
  초기치를 무시할 경우 : 별도로 지울필요없이 원하는 문자를 바로입력한다.

  Esc    : 입력취소
  Cntl-Y : 문장 전체를 지움.
  ←/→  : 좌/우로 한칸 이동.
  Home   : 문장의 처음.
  End    : 문장의 끝.
  BS     : 커서 앞쪽의 문자를 지움.
  Del    : 커서 위치의 문자를 지움.
  엔터   : 문장입력을 마침.

  F10    : MCD로 디렉토리명을 삽입해준다.     (도스명령,명령수정,옵션입력시)
  F11    : QCD 메뉴로 디렉토리명을 삽입해준다.(  "         "        "      )
  Ins    : 선택막대의파일명인 "%F"를 삽입한다.(  "         "        "      )
  ^Ins   : 선택막대의파일명을 직접 삽입한다.  (  "         "        "      )

  S-F1 에서 S-F10 : QCD에 지정된 디렉토리명이 삽입된다. ( "     "     "    )

  위/아래 화살표(도스명령) : 이전에 사용했던 명령을 불러낸다.

   롤 시킨다.   기타 파일을 복사,이동,삭제,... 기타 등등의 작업중에 우측버튼을
   누르면 해당작업이 취소되거나 이전상태로 빠져나간다.

   [참고]  드래그(Drag)는 버튼을 누른상태로 움직이다 떼는 것이고,
           클릭(Click)은 버튼을 가볍게 한번 눌렀다 떼는 것이다.

           메인화면에서와 풀다운 메뉴, MCD에서는 드래그도 지원한다.

  읽기전용이나 시스템파일은 파일크기가 노란색으로 표시되고  숨은파일은 보라색
  으로 표시된다.    파일명이 노란색(.BAT), 밝은하늘색(.COM), 밝은초록색(.EXE)
  으로 표시되는 것은 실행이 가능한 프로그램 파일이며, 그외의 색깔을 가진 파일
  은 사용자가 색깔을 지정한 것이다.   디렉토리는 붉은색으로 표시된다.
     (참고) 프로그램을 포함한 파일의 색깔은 모두 사용자가 바꿀 수 있다.

  CD-ROM드라이브에서는 디스크총용량부분이 "CDROM"으로 표시된다.   정확한 용량
  계산을 하기 위해서는 CD-ROM드라이브의 특성상 시간이  많이 소요되기 때문이며,
  실제총용량은 MCD에서 선택막대를 루트디렉토리에 옮긴후 F9키를 누르면 된다.

  빨리찾기 기능을 사용할 때는 입력되는 내용이 볼륨명이 표시되는 곳에 출력되고
  주석이 있는 파일은 모드1에서는  파일의 오른쪽에 표시되고, 그외의 상태에서는
  Path가 표시되는 부분에 출력된다.

  <참고> 파일용량및 디스크의 용량중 KB(킬로바이트)와 MB(메가 바이트)의 용량은
         다음과 같은 기준으로 계산한다.   컴퓨터에서 사용하는 단위는 일반적인
         수량표시에 사용하는 1000단위가 아니라 1024단위(2의 10승)이다.
         즉, 1kg = 1000g 이지만  1KB는 1000 byte가 아니라 1024 byte이다.
             1MB = 1024KB = 1048576 byte이다. (상식적인 내용~ 착오없기를~)

▶이전상태 : Esc키. 이전의 상태로 돌아가거나 취소하는 키로 사용된다.
             Esc키는 특별하게 MSET에서 기능을 사용자가 지정할 수 있다.

▶도스복귀 : Alt-X

▶환경설정 : Ctrl-F12. MSET.EXE를 실행하여 MdirIII의 환경을 지정한다.

▶메뉴호출 : F12 - 풀다운 메뉴를 호출한다. (Ctrl-Ins키도 메뉴를 호출한다.)
             풀다운 메뉴에서만 사용할 수 있는 기능은 다음과 같으며, 그외의
             기능은 명령키(Hot Key)로도 사용할 수 있다.
            ┌─ ▷ About상자(제작번호와 저작권자, 등록자, 등록번호)
            │   ▷ 펑션키와 확장자에 지정해 놓은 프로그램들 보기
            └─ ▷ 그래픽 문자 바꾸기(한글코드 선택)

▶도스명령 : 슬래쉬(/)키 또는 물음표(?)키 (아래쪽 "도스명령" 참조)
             슬래쉬(/)키는  메모리를 점유하지 않고 실행시키며, 물음표(?)키는
             스와핑을 하여 실행한다. 도스상에서 입력가능한 모든명령을 사용할
             수 있다. 실행후 정지하고 싶으면 마지막에 '%'문자를 추가한다.
             펑션키를 지정할때 사용할 수 있는 모든 변수의 사용도 가능하다.
            ┌─ ↑ / ↓   : 이전에 사용한 명령을 부를수 있다   ──────┐
            │   Ins       : "%F"변수를 삽입해 준다                         │
            │   Ctrl-Ins  : 선택막대의 파일명을 직접 삽입해 준다           │
            │   F10       : 디렉토리명을 MCD로 삽입해 준다                 │
            └─ PgUp/PgDn : 히스토리 메뉴(F12키도 가능)──────────┘

▶도 움 말 : F1 - 간단한 도움말을 본다.
                  M.EXE파일과 같은 디렉토리에 M.HLP파일이 있어야 한다. /J옵션
                  시에는 M-J.HLP가, /K옵션사용시에는 M-K.HLP가 있어야 한다.

                   보기 바란다.

▶ QCD     : F11 - 디렉토리 빨리 바꾸기(Quick Change Directory).  자세한 것은
                   아래쪽의 'QCD'부분을 보기 바란다.


▶실행명령 :┌─ 엔터      : 도스메모리 전부를 사용 (Ctrl-엔터:옵션)────┐
            │   Tab       : 스와핑하여 실행(Shift-Tab:옵션), 확장자별실행. │
            │   Alt-SPACE : M3가 종료되고 실행 (Ctrl-SPACE:옵션)           │
            │                                                              │
            │   펑션키    : 펑션키에 지정된 프로그램을 실행                │
            │                                                              │
            └─ Ctrl-T    : 이전에 실행한 프로그램의 종료화면을 본다.───┘

             Ctrl-SPACE키는  실행파일이 아니더라도  도스명령행으로  파일명을
             입력해주기 때문에 도스명령행을 수정할 수 있는 경우 유용하게  사
             용할 수 있다.

             엔터키나 Tab키는 정의된 확장자에 대한 프로그램을 실행 할 수 있다.
             엔터키로는 압축파일과 관련된 것은 압축파일보기로 넘어가기 때문에
             tab키를 사용해야 한다.

             펑션키는 F2번에서 F9번까지와  Ctrl-F1에서  F10까지,  Alt-F1에서
             F10까지에 자주 사용하는 프로그램을 지정하여 실행시킬수 있다. 펑
             션키에 여러개를  지정한 경우는  선택상자에서 엔터키(그냥실행)나
             Ctrl-Enter키(수정후 실행)를 사용하면 된다.

▶파일작업 : 여러개의 파일을 한번에 작업하고자 할때는 먼저 파일을 선택하고 나
             서 아래의 해당 기능을 작동시키면 된다.  단, 파일보기, 파일찾기는
             선택한 파일과 상관없이 작동한다.

┌ 《복사》  Alt-C  - 파일을 복사한다.  복사되는 디렉토리에 이미 같은 이름의
│           (Ins키)  파일이 존재할 경우는 다음과 같이 물어본다.
│                ┌─OverWrite(덮어씀),All(모두덮어씀), Skip_All(모두통과) ┐
│                └─Skip(통과), Rename(이름바꿔 복사), Newer(새로운것만)─┘
│
│                 ※ 이 메뉴의 아래쪽에는 원본파일과 대상파일의 날짜, 시간,
│                    크기, 디렉토리명이 각각 표시되는데, 날짜/시간/크기가
│                    모두 동일한 것이면 같은 색으로 표시되고 한가지라도 다른
│                    것이 있다면 대상파일에 대한 항목이 강조되어 표시된다.
│
│                    'OverWrite'는 이미 존재하는 파일을 무시하고 덮어 쓴다.
│                    'All'은       이후 모든 파일을 덮어 쓴다.
│                    'Skip_All'은  이미 존재하는 파일은 모두 통과한다.
│                    'Skip'은      현파일 하나만 복사하지 않고 통과한다.
│                    'Rename'은    이름을 바꿔서 복사한다. 바꾸려는 이름또한
│                                  존재할 경우는 다시 물어본다.
│                    'Newer'는 대상디렉토리(Target)의 파일과 시간을 비교하여
│                              새로운 파일일 경우나, 동일한 파일이 없는 경우
│                              에만 복사를 한다.
│
│                    파일을 복사할때 복사되는 디스크(Target)에 용량이 부족할
│                    경우는 먼저 복사를 시작하기 전에 에러메세지를 출력한다.
│                         'Not enough free space for COPY. continue?"
│                    이것은 선택한 파일들이  한번에 복사될수 없음을 사용자에
│                    게 알려준다. 만약 계속 진행(Yes)하면 용량이 되는곳 까지
│                    복사한후 디스크를 바꿀수 있도록 위 메세지가 다시 나오게
│                    된다. 디스크를 바꿨으면 Yes를 선택하면 된다.  위와 같은
│                    방법으로 여러개의 디스크에 나누어 복사할 수 있다.
│
│                    Esc키로 작업을 취소하거나 복사중 에러가 발생했을 경우는
│                    복사가 끝난 파일만 선택표시가 지워진다.
│
│                    파일을 복사/이동할때 4dos/ndos의 주석도 함께 처리해준다.
│                    < 마우스로는 화면하단 상태라인의 파일크기부분 클릭 >
│
│ 《이동》  Alt-M  - 파일을 이동시킨다. 원본은 지워진다.   같은 드라이브내의
│                    파일이동은 매우 빠르게 처리된다.  나머지는 복사하는것과
│                    동일하다.
│
│ 《이름》  Alt-R  - 파일이나 디렉토리의 이름을 바꾼다. 도스의 특수한 용도로
│                    사용되는 파일 이름으로는 바꿀수 없다.
│                ┌─COM1,  COM2,  COM3,  COM4,  NUL,  PRN,  LPT1,  LPT2,─┐
│                └─CON,   AUX  ─────────────────────┘
│                    여러개의 파일을 선택한 후 와일드 카드를 사용하여 이름을
│                    바꿀 수도 있다.
│
│ 《삭제》  Alt-D  - 파일이나 디렉토리를 지운다. 중요한 데이터를 지우지 않도
│           (Del키)  록 조심해서 사용해야 한다. 읽기전용, 숨은파일, 시스템의
│                    속성을 가진 파일을 지울때는 정말 지울지 다시 물어본다.
│                    디렉토리가 비어있지 않을때는 다시 물어본 후 지운다. 단,
│                    루트디렉토리는 드라이브전체의 파일이 모두 삭제되니 주의
│                    해서 사용해야 한다. 루트디렉토리를 지우려고 하면 총 3회
│                    에 걸쳐 확인을 하게 된다. (주의!!!)
│
│ 《보기》  Alt-V  - 파일을 본다.   환경설정 프로그램(MSET.EXE)에서 파일보기
│                    프로그램을 지정해 줄 수 있다.
│
│ 《인쇄》  Alt-P  - 텍스트 파일을 프린트한다.
│
│ 《찾기》  Alt-I  - 파일을 찾는다. 환경설정 프로그램(MSET.EXE)에서 파일찾기
│                    프로그램을 지정해 줄 수 있다.
│
│ 《주석》  @Enter - 주석을 입력한다.  (Ctrl-Alt-Enter글쇠도 동일)
│
│ 《속성》  Ctrl-Z - 파일의 속성을 바꾼다.
│                  ┌─ A키   - Archive  ON/OFF (변경) ──────────┐
│                  │   R키   - ReadOnly ON/OFF (읽기전용)                 │
│                  │   H키   - Hidden   ON/OFF (숨김)                     │
│                  │   S키   - System   ON/OFF (시스템)                   │
│                  │                                                      │
│                  │   엔터  - 현재파일의 속성을 기록한다.                │
│                  └─ End키 - 현재이후 파일전부의 속성을 기록한다. ───┘
└───────── < 마우스로는 화면하단의 상태라인 속성표시부분 클릭 >

▶디스크   : Ctrl-P - 하드디스크를 파킹시킨다. 내장된 파킹기능은 하드디스크가
                      여러개 있어도 모두 파킹시켜준다.  파킹프로그램은 사용자
                      가 지정해 줄 수 있다.
             Ctrl-V - 볼륨명을 바꾼다. 마우스로는 볼륨명표시부분 클릭.
             Ctrl-C - 디스크를 복사한다. (프로그램 지정가능)
             Ctrl-F - 디스크를 포맷한다. (프로그램 지정가능)

             Ctrl-Q - Win95의 긴파일명을 주석으로 바꾸어 준다. 이동형매체(플로
                      피등)나 CD롬 드라이브에서는 주석을 디스크에 기록해 두어서
                      다음번 사용시 사용할 수는 없다. 화면상으로만 볼 수 있다.
                      (아래 Win95부분 참고)

▶선택사항 : Alt-L  - 파일명을 대/소문자로 출력한다.
                      명령행 옵션은 'a'이다.
             Alt-A  - 숨은파일도 함께 출력할지/안할지를 선택한다.    동시에
                      속성을 볼수 있다.
                      명령행 옵션은 'a'이다.
             Alt-Z  - 숨은 파일을 파일리스트에 포함할지/안할지만 선택한다.
                      디스크용량 옆에 'H'가 표시된다. 명령행 옵션은 'h'이다.
             Alt-F  - 파일스펙을 바꾼다. ('필터'라고도 함)
                      경로명을 입력하면 디렉토리를 바꿀수도 있으며,  화면에
                      표시할 파일을 제한할 수 있다.
                      여러개의 파일명을 지정해도 된다.(최대 15개)
                      ┌─ "*.doc"  <- 현재디렉토리의 *.doc파일만 보여줌. ─┐
                      │   "c:\"    <- C:\로 디렉토리 이동함.               │
                      │   "a"      <- 현재디렉토리의 a*.*파일만 보여줌.    │
                      │   "a."     <- 현재디렉토리의 a.파일만 보여줌.      │
                      └─ "*.doc;*.txt;*.gif" = ".doc;.txt;.gif" ─────┘
                      < 마우스로는 화면상단의 "Path c:\xxxx\*.*"부분 클릭 >
             Alt-G  - 실행파일만 보여준다. 명령행 옵션은 'd'이다.
             Ctrl-E - EGA/VGA의 28, 43/50라인 모드로 전환된다. (EGA/VGA에서)
                      명령행 옵션은 'V'(28라인)와 'v'(43/50라인)이다.
             Alt-=  - 메인화면의 파일리스트화면의 크기를 자동조정/고정 선택.
                      명령행 옵션은 '='이다.
             Alt-\  - 메인화면의 파일리스트의 가로구분선을 표시/없앨 수 있다.

      (참고) 명령행 옵션은 도스프롬프트상에서   M3를 실행할때 사용하는 옵션을
             주는 것을 말하는 것이며, 선택사항의 종류는 앞쪽의 '선택사항'부분
             을 참고하기 바란다.

▶기타     : Alt-W     - 작업디렉토리를 지정하거나 현재디렉토리와 맞바꾼다.
             Ctrl-W    - 작업디렉토리를 닫는다.
             Alt-Enter - 주석을 입력/수정/삭제한다.
             Ctrl-S    - 화면보호기능을 가동시킵니다.
             Ctrl-BS   - 이전 디렉토리로 간다. 이 기능은 MCD나 메인화면에서 여
                         러가지 방법으로 디렉토리를 바꾼후  바로 이전의 디렉토
                         리로 돌아가고 싶을 경우나 파일을 복사/이동/압축해제한
                         디렉토리로 들어가 보는 용도로 적합하다. 파일을 복사한
                         후에는 대부분의 사용자가 제대로 되었는지 확인을 해 볼
                         것이라 생각된다.

      (참고) 작업디렉토리 지정은 여러곳에 흩어져 있는 파일을 하나의 디렉토리에
             옮기거나  하나의 디렉토리에서 많은 작업을 해야할 경우 지정하면 편
             리하다. 복사/이동등의 각종작업에서 작업디렉토리가 자동으로 대입된
             다. 작업디렉토리를 지정하게 되면 메인화면의 우측상단에 Work ...라
             는 식으로 지정된 작업디렉토리가 표시된다. 이때 <Alt-W>키를 누르면
             현재 디렉토리와 작업디렉토리를 맞바꿀 수 있다.  작업디렉토리를 닫
             을 때에는 <Ctrl-W>키를 누르면 된다.

 메인화면에서 <F10>키를 누르면 작동된다. ( MdirIII Change Directory )
 디렉토리구조를 알아보기 쉽게 계층구조로 보여주며, 쉽고 빠르게 원하는 디렉토리
 로 이동하거나 작업도중 디렉토리를 선택할때에 사용된다.

 한번 읽어들인 트리구조는 파일로 저장해 두어 다음번 사용시 더 빠르게 MCD화면이
 나타나게 되고, 디렉토리 구조가 실제와 다르면 사용도중 재검색을 하게 된다.
 단 자동 재검색기능은  현재 디렉토리가 트리구조 파일에 포함되어 있지 않을 경우
 에만 자동으로 행한다.  따라서 트리구조에 포함되지 않은 디렉토리가 있을 경우는
 메인화면에서 트리구조에 포함되지 않은  디렉토리로 이동한후 <F10>키를 눌러 MCD
 로 들어가면 된다. 아니면 MCD상태에서 강제로 전체디렉토리 구조를 다시 검색하게
 할 수 있다.(F2키로)

            ┌─ Esc    : 빠져나가기/기능취소 ───────────────┐
            │   ↑     : 위쪽의 형제 디렉토리로 선택막대 이동              │
            │   ↓     : 아래의 형제 디렉토리로 선택막대 이동              │
            │   ←     : 부모 디렉토리로 선택막대 이동                     │
            │   →     : 자식 디렉토리로 선택막대 이동                     │
            │   ^←    : 무조건 위쪽(혹은 앞쪽) 디렉토리로  선택막대 이동  │
            │   ^→    : 무조건 아래쪽(혹은 뒤쪽) 디렉토리로 선택막대 이동 │
            │   Home   : 루트   디렉토리로 선택막대 이동 (/키도 같은 역할) │

 CD-ROM 드라이브의 경우 디렉토리의 구조를 임시디렉토리에 저장해 놓는데, 각각의
 CD타이틀별로 별도의 파일명을 사용하게 된다. 트리구조 저장파일의 이름은 Mdir이
 임의로 작성하게 되며 확장자는 .MCD이다. 단, CD타이틀에 정확한 트리구조 파일이
 있을 경우는 그것을 이용하며 그렇지 않을 경우는 한번 재검색 한다.

            ☞ 디렉토리크기보기(F9키) - 현재디렉토리와 자식디렉토리의 크기

            │   Free space       : 디스크의 사용가능한 용량
            └─ Allocation unit  : 파일의 최소 할당 단위

   디렉토리 빨리 바꾸기(QCD)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

   10개의 디렉토리를 사용자가 지정해 놓고 필요할 때 바로 이동할 수 있는 기능이
   다. 이름하여 '디렉토리 빨리 바꾸기(Quick Change Directory)'라고 부른다.

   하드 디스크에는 수백개 이상의 디렉토리가 있겠지만 실제로 자주 들어가는 디렉
   토리는 몇개가 되지 않는다. 이런 발상에서 이 기능은 만들어졌다.

   QCD는 두가지 방법으로 사용할 수 있다. 첫째 메인화면이나 MCD상태에서 <F11>키
   를 눌러 메뉴방식으로 사용하는 경우이고, 둘째는 <Shift+펑션키>를 눌러 한번에
   이동하는 방법이다.

   QCD에 디렉토리를 지정할 경우는 메인화면에서 <F11>키를 누른후 <Ctrl-Enter>키
   를 누르면 MCD화면이 나타나게 되고 이곳에서 고른 디렉토리가 지정된다. 지정한
   디렉토리는 자동으로 기억된다.

   QCD에 지정된 항목을 지울 경우는 <F11>키를 눌러 QCD메뉴를 부른 다음 필요없어
   진 항목에서 <Del>키를 누르면 된다.

   메인화면이나 MCD상태, 그리고  드라이브 선택상자(MCD에서 F3키), 도스명령에서
   QCD기능을 사용할 수 있다. 메인화면과 MCD, 드라이브 선택상자에서는 Shift키를
   누르면  화면의 상단 혹은 하단에 QCD에 지정된 디렉토리를 보여준다.

   각각 지정된 10개의 디렉토리는  <Shift-F1>에서 <Shift-F10>키에 배정되어 있다.
   단, MCD에서는 <Shift-F1>에서 <Shift-F10>키를  두번 눌러야 디렉토리가 선택된
   다. 한번 누를 경우는 선택막대만 옮겨지는데 이는 실수를 방지하기 위한 것이다.
   하지만 메인화면에서는 한번에 바로 디렉토리가 바뀐다.

   도스명령(Dos command)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

   도스 프롬프트(Prompt)상에서 입력할 수 있는 모든 명령을 MdirIII의 도스명령기
   능을 사용하게 되면 이용할 수 있다. 도스명령은 메인화면에서 </>키 혹은 <?>키
   를 누르면 사용할 수 있다. </>키는 메모리를 전혀 점유하지 않고 실행하는 방법
   이고 <?>키는 스와핑을 하여 실행하는 방법이 다를 뿐이다.

   도스프롬프트상에서  명령을 입력할 때의  불편함은  다들 알다시피 한번 입력한
   명령의 수정이 힘이들고, 이전에 사용한 명령을 다시 사용할 때의 제약이 따른다.
   따라서 매번 같은 명령을 입력하는 수고를 M3에서는 할 필요가 없게 된다.  또한
   한번 사용한 명령은  컴퓨터의 전원을 내려도 기억되어 있기때문에  다음에 다시
   사용할 때 매우 편리하다. 최대 20개까지의 도스명령이 기억된다.

   또한 MdirIII에서 사용가능한  모든 변수(!.!등)를 사용할 수 있으며, <F10>키를
   눌러 MCD로 디렉토리명을 대입할 수도 있다. 도스프롬프트에서 프로그램을 실행
   할때보다 훨씬 강력한 환경을 제공하는 셈이며, 활용은 사용자에게 달려있다.

   처음에 도스명령 상태로 들어올때에  파란색 배경으로 나타나는 글자는 초기치를
   그대로 사용할지 수정할지를 물어보는 상태이다.    즉, 그대로 사용할 것이라면
   그냥 <엔터>키를 누르면 되고, 새로 입력할 경우는 지울 필요없이 그냥 입력하면
   된다.   덧붙여서 입력하거나 수정을 하려고 한다면 좌/우 화살표키나 <Home>키,
   <스페이스>키등을 눌러준 후 필요한 사항을 입력하면 된다.  이전에 사용한 명령
   을 다시 사용하고자 할 경우는 상/하 화살표키를 누르면 된다.

   예전에 사용하던 명령중 앞의  몇글자만 입력한후 상/하 화살표키를 누르게 되면
   예전에 사용하던 명령중 앞부분이 동일한 명령을 바로 불러내어 준다.

   이때 한가지 짚고 넘어갈 것은 파란색으로 명령이 반적되어 나올때에 상/하 화살
   표키를 누르게 되면  차례대로 이전에 사용하던 명령이  하나씩 불려져 나오지만
   반전되어 있지 않을때는 지금입력되어 있는 명령과 앞부분이 동일한 명령만 찾아
   보게 된다는 점이다. 이때는 전부 지운후 위/아래 화살표키를 누르면 된다.

   설명서만 보고 있지 말고 직접 하나씩 해보는 방법이 가장 빠를것이다.

   ▶ 편집에 사용되는 글쇠는 다음과 같다. 여느 워드 프로그램에서 사용하는 편집
      글쇠와 다를것이 없다. 한번 익혀두면 두고 두고 사용할 수 있으리라.

      전부 지우기 : <Ctrl-Y>
      커서 이동   : <Left> / <Right> / <Home> / <End>
      글자 지우기 : <Del> / <백스페이스>
      입력취소    : <Esc>
      입력끝      : <Enter>


   ▶ 이전에 사용하던 명령을 부르는 키는 다음과 같다.

      가장   최근에 사용했던 명령 : <Up>
      가장 마지막에 사용했던 명령 : <Down>
      히스토리 메뉴               : <PgUp>, <PgDn>, <F12>
                                    바로 실행 - <엔터>   수정 - <Ctrl-엔터>


   ▶ 특수기능

      <F10>          : MCD로 디렉토리명을 선택하여 삽입
      <F11>          : QCD 메뉴로 디렉토리명을 선택하여 삽입
      <S-F1>~<S-F10> : QCD에 지정된 디렉토리명 직접 삽입
      <Ins>          : 선택막대 위치의 파일명을 삽입해 주는 변수 %F를 삽입
      <Ctrl-Ins>     : 선택막대 위치의 파일명을 직접 삽입


  이들을 잘 활용하면 도스프롬프트상에서 보다 훨씬 편하게 도스명령을 사용할 수
  있게 된다.

   주석(Description)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

   주석(설명문)이란 8자의 파일명과 3자의 확장자로는 쉽게 구분이 되지 않는 많은
   파일들에 사용자가 간단히 설명을 달 수 있는 기능이다.주석은 파일을 복사/이동
   /삭제/이름바꾸기에서도 따라다닌다. (등록판에서만)

   M3가 처리하는 주석은 4DOS/NDOS(이하 4DOS)의 주석과 호환이 되며, 4DOS에서 생
   성한 주석을 M3가 인식하며 그 반대도 가능하며, 4DOS가 없어도 주석기능은 사용
   할 수 있다. 한 디렉토리내에서 최대 1000개의 주석을 처리할 수 있으며, 주석의
   길이는 제한이 없다.    화면에 표시하는 주석의 길이와 입력/수정시에는 실제의
   주석이 완전히 표시되지 않을 수 있다. 하지만 보통 주석당 40자까지만 사용하며
   길어도 80자 이하이므로 사용시의 제약은 거의 없다고 볼 수 있다.

   물론 남아있는 도스메모리의 양이 부족할 경우는 실제처리가 가능한 주석의 수가
   1000개 이하일 수도 있다. 디렉토리내에 1000개이상의 주석이 있어도 단지 볼 수
   없을 뿐, 파일을 복사/이동/삭제/이름바꾸기등의 작업에서는 아무런 영향을 받지
   않고 완벽하게 처리해 준다. 이러한 제약은 M3의 메모리절약을 위한 것과 화면에
   표시할 공간이 부족한 이유 때문이다.

   4DOS는 디렉토리내에  DESCRIPT.ION이란 파일의 숨은파일(Hidden)을 생성하여 주
   석을 처리한다.   따라서 주석을 달아놓은 디렉토리에는 반드시 이러한 파일들이
   생성될것이다. 바이러스가 아닌가 의심하여 지워버리지는 마시고~

   주석을 입력/수정/삭제하는 명령키는 <Alt-Enter>키이다. 주석을 전부 삭제할 때
   는 DESCRIPT.ION이란 파일을 지워버리는 편이 빠르다.물론 <Alt-Enter>키를 눌러
   하나씩 지워도 상관은 없지만... 입력/수정시 최대 80자까지 편집이 가능하다.
   여러개의 파일을 선택하여 주석을 입력할 수도 있다.

   M3는 주석이 있을 때 자동으로 주석보기 상태로 전환이 되는 기능을 가지고 있다.
   주석을 볼 수 있는 상태는 모드 1(Alt-1)뿐이다. 자동전환 되는것이 싫다면 MSET
   에서 주석보기를 '켜짐'로 해놓으면 화면 상단의 Path 표시부분에 현재 선택막대
   위치의 파일에 대한 주석을 표시해 준다.

   화면보호(Screen saver)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

   일반용에서는 화면보호 프로그램을 지정해도 작동하지 않는다.    그렇게 이쁘고
   멋있지는 않아도 내장된 화면보호기능을 사용할 수 있다.    화면보호프로그램을
   지정했더라도 메모리가 부족하거나  지정된 화면보호 프로그램을 실행할 수 없을
   경우에는 내장된 화면보호기능이 작동된다.

   화면보호기능에 램상주 프로그램을 지정하면 안된다.  일반적으로 "보라"를 지정
   하여 사용하면 무난할 것이며, 기타 공개자료실등에서 쉽게 구할수 있을것이다.

   화면보호의 필요성에 대한 설명은 "보라"의 설명서를 참고하시기 바라며, 파일과
   디렉토리의 복사/이동기능,파킹등을 제외한 대부분의 기능사용중에 대기시간동안
   키입력이나 마우스버튼이 눌려지지 않으면 화면보호를 시작하게 된다.

   압축파일(Archive)
▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥▥

 압축파일에 선택막대를 옮겨 놓고 엔터키를 누르면 압축파일 보기상태가 된다.
 이 때는 선문자의 색깔이 보통때와는 달라지며, 화면상단에 압축파일명이 표시된
 다. 괄호안에는 정확한 압축방식이 표시된다. 압축파일의 확장자를 잘못준 것이
 나 SFX일 경우 참고하면 되겠다.

 하위디렉토리까지 압축된 파일의 경우는 주석으로 압축파일내에서의 정확한 디렉
 토리명이 표시된다. 디렉토리 재현기능이 켜져있을때는 하위디렉토리를 포함하여
 압축된 파일은 디렉토리구조를 그대로 재현해서 보여준다.

 압축파일이 비정상적일 경우는 'Broken file'이란 메세지가 표시된다.

 . 빠지기 : <Esc>키는 그냥 압축파일보기를 빠져나간다. 혹은 <EXIT>를 선택.
            <.>키는 이전 단계로 빠져나간다.  <\>키는 압축파일의 루트디렉토리
            로 빠져나간다. 마우스로는 화면상단의 "Archive T.LZH (LZH)"부분을
            클릭하면 빠져나간다.
 . 전  환 : 디렉토리 재현기능을 <Alt>-<`>키로 ON/OFF시킬 수 있다.
            ON일 경우는 <EXIT>로 OFF일때는 <Exit>로 대/소문자로 구분하여
            화면에 표시되므로 쉽게 현재 상태를 알 수 있다.
            ON일 경우는 디렉토리를 그대로 재현해서 보여주고 OFF 일때는 압축
            파일속의 파일들을 디렉토리에 상관없이 모두 보여주며 주석으로 정
            확한 디렉토리명을 보여준다.
 . 풀  기 : 파일을 선택(SPACE)한 후 Ctrl-X키를 누른다.
 . 지우기 : 파일을 선택(SPACE)한 후 Alt-D키를 누른다.
 . 실  행 : 응용프로그램이나 정의된 확장자위에서 엔터키나 Tab키를 누른다.
            펑션키로 지정한 프로그램도 실행이 가능하다. 프로그램명뒤에 !.!나
            %F 혹은 !\!.!를 포함하면  해당파일의 압축을 임시로 풀어서 실행할
            수 있다.
 . 도스명령 : /키를 눌러 도스명령을 사용할 수 있다. (?키도 /키와 동일)
 . 파일보기 : Alt-V키를 눌러서 해당파일을 볼 수 있다.

 프로그램을 실행하는 경우 실행되는 프로그램이 디렉토리를 바꾸고 종료해도  원
 래의 디렉토리로 돌아온다. 압축파일보기중에는 메모리를 전혀 점유하지 않고 실
 행할수 있는 기능을  사용할 수 없게 된다. (엔터키를 눌러도 Tab으로 작동되고,
 <?>키를 눌러도 </>키로 작동된다. "%B"변수는 무의미해진다. )

 c:\iyagi\down\x.zip라는 식으로 이름을 줘도 된다.

 위에 옮긴후에 압축을 하면 됩니다. (ARJ등의 경우)

        서브디렉토리까지 압축된 TEST.ZIP파일을 풀고 싶을때.
            -> 일반적인 압축풀기와 동일하다.

        ARJ로 용량에 맞게 나누어 A:에 TEST.ARJ로 분할하여 압축하고플때.
            -> 압축파일명을 입력할때 "-va a:test.arj"를 입력해준다.
        ARJ로 멀티볼륨으로 압축된 TEST.ARJ파일을 풀고 싶을때.
            -> "-v *.*"를 파일스펙을 입력하는 상자에서 입력해준다.
               이때 선택막대가 TEST.ARJ파일위에 있아야 한다.

 ▨ 긴파일명 ▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨

 Win95에서 MdirIII를 사용하게 되면 긴 파일명을 지원한다. 화면에서는 기본적으로
 짧은 파일명을 사용하게 된다.

 하지만 이와는 별개로 파일이나 디렉토리를 복사/이동하거나 삭제 혹은 이름을 바
 꾸게 되면 긴파일명을 파괴하지 않고 원래의 긴파일명을 유지하게 된다. 화면상에
 서 긴파일명을 바로 지원하지 않는 이유는 과거도스와의 호환성문제도 있겠지만
 프로그램을 단순화시키기위한 의도이다. 그래서 생각한 것이 긴파일명을 주석으로
 바꿔주는 기능을 추가한 것이다.

 긴파일명을 확인하고 싶을 경우는 간단히 <Ctrl>-<Q>키를 눌러주면 되는것이다.
 한번 변환된 것은 디스크에 기록해 두기 때문에 다음번에는 바로 확인이 가능하다.
 단 이동형매체(플로피디스크)나 CD롬 드라이브에서는 디스크에 주석을 기록해 두지
 않는다. 또한 다른 주석이 이미 달려 있는 파일은 변환되지 않는다.
 과거 버젼의 사용법은 그대로 유지하면서 새로운 기능을 적절히 추가하기위함이다.

 하지만 MS-DOS 6.22이하의 버젼에서는 긴파일명을 가진 파일을 복사하거나 이동하
 는 등의 작업에서 긴파일명을 계속 유지할 수 없다.

 ▨ 주석입력(Alt-Enter)  ▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨

 주석을 입력할 때 사용하는 <Alt>-<Enter> 글쇠는 Win95나 Win 3.1에서 도스창을
 확대/축소하는 단축키로 예약되어 있다. 따라서 도스창의 등록정보를 수정해주거
 나 <Ctrl>-<Alt>-<Enter>키로 사용하면 문제가 없다.

 비슷한 문제로 <Alt>-<SPACE>키로 M을 종료하면서 프로그램을 실행하는 기능도
 <Ctrl>-<Alt>-<SPACE>키로 대체하면 문제가 없다.

 ▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨▨

 그리고 /x를 인자로주어 풀메모리 실행기능이 작동하지 않도록해야만 프로
 그램 실행시 종료되어 버리는 문제를 막을 수 있다. 배포되는 M.PIF는 /x
 옵션이 포함되어 있으므로 그대로 사용하여도 무방하다.


 Win95의 도스창에서 사용할 경우는 Win95용 프로그램이나 도스용프로그램을 모두
 실행시킬 수 있다. 때문에 펑션키나 확장자 지정할때에 Win95용 프로그램을 사용
 할 수 있다. 그리고 윈95의 익스플로러에 등록된 확장자에 대한 실행은 다음과
 같이 할 수 있다. "START 파일명" START.EXE는 WIN95가 설치된 폴더의 COMMAND폴
 더 안에 있다. 도스상에서 Win95의 프로그램을 보다 편리하게 실행할 수 있도록
 하는 명령이다. 예를 들어 "start test.wav"라는 명령을 실행하면 Win95의 매체
 재생기가 실행되어 test.wav라는 파일을 연주할 것이다. 펑션키나 확장자 지정시
 이를 응용할 수 있을 것이다. (특정 펑션키나 확장자에 이를 등록해 두면...)
